# Contributing Guide

## Purpose

This document provides general guidelines for contributing to the Substracker project.

---

## Development Workflow

1. Clone the repository.
2. Create or switch to the working branch.
3. Implement the required changes.
4. Test the application.
5. Commit changes using clear commit messages.
6. Push changes to the remote repository.

---

## Commit Message Format

Recommended format:

```
type: short description
```

Examples:

```
docs: update README
feat: add subscription validation
fix: resolve login issue
refactor: improve widget structure
```

---

## Code Style

- Follow Dart formatting conventions.
- Use meaningful variable names.
- Keep widgets reusable.
- Separate UI and business logic whenever possible.

---

## Repository Rules

- Keep commits focused on a single purpose.
- Write clear commit messages.
- Avoid committing unnecessary generated files.
- Update documentation when adding new features.
